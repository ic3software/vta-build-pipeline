Programmatic clients should call POST /v1/auth/challenge then POST /v1/auth/ to mint a bearer JWT. The browser-based passkey flow that mints a cookie session lands in a follow-up.

This is the MVP admin shell. It ships as in-tree HTML/CSS/JS so cargo build is self-contained — no npm, no node, no network. Operators wanting a richer UX can replace the files under vtc-service/admin-ui/ and rebuild.
